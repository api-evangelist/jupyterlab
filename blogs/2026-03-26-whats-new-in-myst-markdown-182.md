<p>The Jupyter Book team just released <strong>mystmd 1.8.2</strong> and a new version of the MyST react theme! Below are a few things that stand out.</p><h3>Interactive widgets with the {anywidget} directive</h3><p>You can now <a href="https://mystmd.org/guide/widgets">embed JavaScript widgets</a> in your MyST pages using the new {anywidget} directive. Widgets follow the <a href="https://anywidget.dev/">anywidget specification</a>, which gives plugin authors an extension point for embedding JS capabilities in their documents.</p><p>A widget is a JavaScript module that exports a render function. You pass it initial state as JSON in the directive body, and it renders an interactive element on the page. You could use this for interactive figures, explorable explanations, or custom visualizations without leaving MyST.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/0*lkUWCUNKiKhXcUUK.png" /><figcaption>A little confetti demo that we use to show off embedding widgets in a MyST document with anywidget!</figcaption></figure><p><strong><em>Note</em></strong><em>: Widget support is experimental. The interfaces may change as we learn more about usage patterns. Only load widgets from sources you trust, since they execute JavaScript in the reader’s browser!</em></p><h3>Table of contents improvements</h3><p>The {toc} directive now supports a <a href="https://github.com/jupyter-book/mystmd/pull/2705">children context option</a>, which lets you display only the child pages of the current page rather than the full project table of contents. Use it on landing pages and index pages that introduce a section and link to its sub-pages. You can see an example in the <a href="https://jupyterbook.org/latest/get-started/">Jupyter Book getting started guide</a>.</p><h3>Better link handling across multi-site projects</h3><p>If your project spans multiple domains (e.g., a docs site and a blog), you can now <a href="https://github.com/jupyter-book/myst-theme/pull/816">configure a domain to be treated as “internal”</a> for link styling. Links to that domain will look like in-site navigation instead of external links. Check out <a href="https://jupyterbook.org/latest/get-started">the Getting Started guide</a> for an example.</p><h3>Upgrade</h3><p>To get the latest version of mystmd:</p><pre>npm install -g mystmd</pre><p>or for pip users:</p><pre>pip install -U mystmd</pre><p>The theme updates automatically. Delete your _build folder and mystmd will download the latest theme on your next build.</p><h3>Changelogs</h3><p>Find an ongoing list of releases in the Jupyter Book ecosystem here:</p><p><a href="https://jupyterbook.org/releases">jupyterbook.org/releases</a></p><p>You can also read the original <a href="https://github.com/jupyter-book/mystmd/releases/tag/mystmd%401.8.2">mystmd release notes</a> and <a href="https://github.com/jupyter-book/myst-theme/releases/tag/myst-to-react%401.1.4">myst-theme release notes</a>.</p><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=b7ae8975b03e" width="1" /><hr /><p><a href="https://blog.jupyter.org/whats-new-in-myst-markdown-1-8-2-b7ae8975b03e">What’s New in MyST Markdown 1.8.2</a> was originally published in <a href="https://blog.jupyter.org">Jupyter Blog</a> on Medium, where people are continuing the conversation by highlighting and responding to this story.</p>

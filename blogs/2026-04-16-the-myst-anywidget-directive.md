<p>Authors of MyST Markdown can now embed interactive JavaScript widgets directly in content using the new {anywidget} directive.</p><h3>What this means</h3><p>You can now add any javascript interactivity to your MyST Markdown website or Jupyter Book.</p><p>Here’s an example that creates a clickable button 👇 to see an interactive version check out <a href="https://jupyterbook.org/blog/posts/2026/the-myst-anywidget#how-we-got-here">the post on the Jupyter Book blog</a>.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*TksTxGw_ahJkRxlsIVU5_g.png" /></figure><h3>How we got here</h3><p>The idea of a portable widget interface for interactive computing isn’t new: <a href="https://anywidget.dev">anywidget</a> has been an emerging standard in the Jupyter community for a while, leveraging modern Javascript principles (via <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Modules">ESM Modules</a>), and giving you a simple initialize({ model }); render({ model, el })<a href="https://jupyterbook.org/blog/posts/2026/the-myst-anywidget#fn-model">[1]</a> contract so that widgets can be written once and reused across notebooks. AnyWidget in Jupyter interfaces supports tight integration with the kernel, allowing interactions similar to ipywidgets<a href="https://jupyterbook.org/blog/posts/2026/the-myst-anywidget#fn-jupyter">[2]</a>.</p><p>From the MyST Markdown and Jupyter Book point of view, we’ve focussed on supporting the render() side of the AnyWidget interface (see <a href="https://anywidget.dev/en/afm/#what-is-afm">AnyWidget Frontend Modules</a>) to allow MyST users to bring any kind of javascript-based interactivity into their articles and books <em>without having to lean on Jupyter at all</em>.</p><p>That part of the story started at SciPy 2024 with <a href="https://github.com/manzt">Trevor Mantz</a> and Steve Purves hacking through a proof of concept during the sprints. Curvenote built working support for <a href="https://www.npmjs.com/package/@curvenote/any-widget">AnyWidgets as a MyST Markdown extension</a> while working with researchers to support domain specific visualizations, and then upstreamed the implementation to create the {anywidget} directive that has just been released in mystmd and jb2.</p><p>The new directive in mystmd and the respective supporting package in @myst-theme/anywidget evolved with input from the JupyterBook team. Here’s how it is currently structured:</p><ul><li>anywidget is a new node in the MyST AST, meaning first-class support for this capability outside of notebooks.</li><li>mystmd will bundle ESM and CSS modules at build time, ensuring dependencies are packaged with the book/article when it is published or deployed.</li><li>The ESM and CSS modules can either be (a) hosted remotely (which makes sense for shared widgets that many people are using or collaborating around) or (b) added as local files (which makes sense for widgets specific to a single book/website).</li><li>The NodeRenderer part of the release is an independent package <a href="https://www.npmjs.com/package/@myst-theme/anywidget">@myst-theme/anywidget</a> that can be optionally adopted by theme developers (it’s already built into the core themes).</li><li>For mystmd and Jupyter Book users, to upgrade to the latest theme. Run myst clean --templates before you next start your server and the latest version will be downloaded.</li></ul><h3>Usage</h3><p>From an author’s perspective, point the directive at an ESM module (either a URL to a hosted script or a local path) and the widget runs in the page with its own state and DOM. You may optionally pass in a CSS URL/path and a JSON body of props to initialize the widget model.</p><p>To understand how to build your own widgets, the <a href="https://mystmd.org/guide/widgets">MyST widgets guide</a> walks through the render({ model, el }) signature, styling with Shadow DOM, and cleanup on unmount. It’s the same mental model as anywidget in Jupyter, so if you’ve written or seen Jupyter AnyWidgets, you’re already most of the way there.</p><p>If you want to try it, the <a href="https://github.com/jupyter-book/example-widgets">example-widgets repo</a> has small demos (confetti, div-map, etc. Contributions are welcome!), and the <a href="https://opensci.dev/">opensci.dev blog</a> has some live examples using scientific datasets.</p><h3>What’s next</h3><p>Widget support in MyST is still marked experimental so the details may evolve. There are still some things on the roadmap for {anywidget} which will no doubt expand as we hear of new requirements from widget creators, but we expect the following to be close to the top of the stack:</p><ul><li>The model part of the interface, which will allow multiple instances of a widget to communicate on the page<a href="https://jupyterbook.org/blog/posts/2026/the-myst-anywidget#fn-model">[1]</a></li><li>Shipping additional dependencies like static files that the widgets may need</li></ul><p>The JupyterBook team is actively working on improving and evolving the widget model and integrating with Jupyter. We look forward to seeing what the community builds — visit the <a href="https://discord.mystmd.org/">discord</a> to showcase what you have built!</p><h3>Acknowledgements</h3><p>A shout out to my co-contributors on this work; to <a href="https://github.com/manzt">@mantz</a> for the original hacking, <a href="https://github.com/rowanc1">@rowanc1</a> and <a href="https://github.com/fwkoch">@fwkoch</a> for iterating on the downstream code, <a href="https://github.com/agoose77">@agoose77</a> for working with me on the upstream changes, <a href="https://github.com/choldgraf">@choldgraph</a> for insightful reviews and a docs! and <a href="https://github.com/jupyter-book">the broader Jupyter Book Team</a> for all the reviews and feedback.</p><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=daa55c348ab2" width="1" /><hr /><p><a href="https://blog.jupyter.org/the-myst-anywidget-directive-daa55c348ab2">The MyST AnyWidget Directive</a> was originally published in <a href="https://blog.jupyter.org">Jupyter Blog</a> on Medium, where people are continuing the conversation by highlighting and responding to this story.</p>
